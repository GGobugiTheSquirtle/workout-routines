# Bodyweight Workout App — Design Spec

> 2026-04-04 | GitHub Pages + Google Sheets | 단일 HTML

---

## Executive Summary

| 항목 | 내용 |
|------|------|
| Feature | 맨몸운동 커리큘럼 & 트래킹 앱 |
| 시작일 | 2026-04-04 |
| 목표 | 초급→고급 단계적 맨몸운동 커리큘럼 + 계획/체크/기록 올인원 |
| 기술 스택 | 단일 index.html (GitHub Pages) + Google Sheets (데이터) + Apps Script (API) |
| 디자인 | 프리미엄 모던, 시스템 다크/라이트 자동 전환, taste-skill 적용 |

### Value Delivered

| 관점 | 내용 |
|------|------|
| Problem | 운동 초급자가 체계적 커리큘럼 없이 방황, 기록/진급 관리 어려움 |
| Solution | 과학 기반 5레벨 프로그레션 + 월간 스케줄 자동생성 + 세트별 기록 |
| Function UX Effect | 앱 열면 오늘 운동 즉시 표시, 원탭 세트완료, 날짜별 일지 자동 축적 |
| Core Value | 운동과학자+트레이너 에이전트팀이 설계한 커리큘럼으로 안전하고 효율적인 성장 |

## 1. 아키텍처

### 1.1 시스템 구조

```
index.html (GitHub Pages)
    │ fetch
    ▼
Google Apps Script (Code.gs)
    │ SpreadsheetApp
    ▼
Google Sheets (4 시트)
```

### 1.2 설계 원칙

- **온라인 우선**: PWA/ServiceWorker/오프라인 캐시 없음. 앱 열면 바로 Sheets에서 fetch → 렌더링
- **앱 셸 분리**: index.html에 운동 데이터 하드코딩 없음. 순수 UI/로직만. 커리큘럼 수정은 시트만 편집
- **LocalStorage 최소 사용**: 세션 중 임시 상태(현재 진행 중인 세트 등)에만

### 1.3 데이터 플로우

- **읽기**: 앱 로드 → Apps Script doGet → Sheets 데이터 반환 → 렌더링
- **쓰기**: 세트 완료 버튼 → Apps Script doPost → logs 시트에 행 추가
- **월간 스케줄**: Claude Code 에이전트팀 → Sheets programs 시트에 직접 작성

## 2. Google Sheets 스키마

### 2.1 exercises (운동 DB)

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| id | string | 고유 식별자 | `pushup_incline` |
| name | string | 한글 운동명 | 인클라인 푸쉬업 |
| category | enum | push / pull / legs / core / swing | push |
| equipment | string | 필요 장비 | 푸쉬업바 |
| level | number | 난이도 1~5 | 1 |
| muscles_primary | string | 주동근 | 대흉근, 전면삼각근 |
| muscles_secondary | string | 보조근 | 삼두근 |
| form_guide | string | 폼 핵심 포인트 | 손 위치는 어깨너비... |
| common_mistakes | string | 흔한 실수 | 허리 처짐, 팔꿈치 벌림... |
| safety_notes | string | 안전 주의사항 | 손목 통증 시 중단... |
| progression_from | string | 이전 단계 운동 id (null=시작) | null |
| progression_to | string | 다음 단계 운동 id (null=최종) | `pushup_standard` |
| video_url | string | 참고 영상 URL (선택) | https://youtu.be/... |

### 2.2 programs (월간 스케줄)

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| date | date | 날짜 | 2026-04-04 |
| day_label | string | 표시명 | Day A — Push+Core |
| order | number | 운동 순서 | 1 |
| exercise_id | string | exercises 참조 | `pushup_incline` |
| sets | number | 세트 수 | 3 |
| reps | string | 목표 횟수/시간 | 10 또는 30초 |
| rest_sec | number | 권장 휴식 (참고용) | 60 |
| promotion_criteria | string | 진급 조건 | 3x15 달성 3회 |
| note | string | 코치 메모 | 지난주 대비 +2rep |

- Claude Code 에이전트팀이 월초에 30일분 행 작성
- 유저 요청 시 Claude Code로 해당 주/일 행 수정
- 휴식일은 day_label = "REST"로 표시, 운동 행 없음

### 2.3 logs (운동 기록)

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| date | date | 날짜 | 2026-04-04 |
| exercise_id | string | 운동 | `pushup_incline` |
| set_number | number | 세트 번호 | 1 |
| reps_done | string | 실제 수행 | 12 또는 30초 |
| note | string | 메모 (선택) | 마지막 2개 힘들었음 |

- append-only: 세트 완료마다 1행 추가
- 수정/삭제 없음 (일지 특성)

### 2.4 user_config (설정)

| key | value | 설명 |
|-----|-------|------|
| current_level | 1 | 현재 레벨 |
| start_date | 2026-04-04 | 시작일 |
| sheets_url | (웹앱 URL) | Apps Script 배포 URL |
| user_name | (이름) | 표시용 (선택) |

## 3. UI/UX 설계

### 3.1 전체 구조

- 단일 HTML, 하단 탭 4개로 네비게이션
- 모바일 우선 반응형 (주 사용 환경: 스마트폰)
- 시스템 다크/라이트 모드 자동 전환 (`prefers-color-scheme`)
- taste-skill 적용 프리미엄 디자인

### 3.2 탭 구성

| 탭 | 역할 |
|---|------|
| **오늘** | 오늘의 루틴 실행. 전체 운동 목록 + 현재 하이라이트 + 세트 체크 |
| **커리큘럼** | 5레벨 로드맵, 프로그레션 체인, 진급 프로그레스 |
| **기록** | 월간 캘린더 + 날짜별 운동 일지 상세 |
| **가이드** | 운동별 풀 가이드 (폼/근육/안전/프로그레션) |

### 3.3 오늘 탭 — 워크아웃 리스트 UX (핵심)

**설계 원칙**: 전체 목록 항상 표시, 현재 운동 하이라이트

- 앱 열면 오늘 날짜로 `programs` 필터 → 4-5개 운동 전체 표시
- 스크롤 없이 한눈에 (30분 루틴이라 종목 적음)
- 각 운동 카드에: 운동명 + 세트/횟수 + 주동근 + 세트 진행률 (✓ ✓ ○)

**현재 운동 하이라이트**:
- 악센트 컬러 보더 + `◀ NOW` 뱃지
- 미완료 세트가 있는 첫 번째 운동이 자동으로 "현재"

**세트 완료 인터랙션**:
- 현재 운동의 "완료" 버튼 탭 → 1세트 완료
- 즉시 Sheets logs에 기록 (doPost)
- 세트 인디케이터 ○→✓ 전환
- 해당 운동 모든 세트 완료 → 다음 운동으로 NOW 이동
- 모든 운동 완료 → "오늘 운동 완료!" 축하 메시지

**상태별 시각 처리**:
- 완료된 운동: 체크마크 + 텍스트 흐려짐 (opacity 0.5)
- 현재 운동: 악센트 보더 + 풀 불투명
- 미진행 운동: 살짝 투명 (opacity 0.7)

**추가 기능**:
- 운동명 탭 → 가이드 탭 해당 운동으로 이동
- 세트 완료 시 메모 입력 (선택, 작은 입력창)
- 휴식일이면: "오늘은 휴식일입니다. 충분히 쉬세요." 메시지

### 3.4 커리큘럼 탭

- 5레벨 전체 로드맵 (아코디언 펼침)
- 현재 레벨 강조, 미도달 레벨 🔒 + 흐리게 (탭하면 미리보기)
- 카테고리별 프로그레션 체인 표시 (push: 인클라인→스탠다드→다이아몬드→...)
- 진급 프로그레스 바: logs 기록 기반 달성도 자동 계산
- 하단 진급 제안 메시지 (혼합 방식: 앱 제안 + 유저 결정)
- "레벨 업" 버튼: 유저가 직접 누를 때만 진급

### 3.5 기록 탭 — 운동 일지

- **미니 캘린더**: 월간 뷰, 운동한 날에 dot 표시, 날짜 탭 → 상세
- **일별 상세**: 해당 날짜의 모든 운동 × 세트별 실제 횟수 + 메모
- **볼륨 요약**: 하단에 총 횟수/시간
- **계획 vs 실제**: programs(목표) vs logs(실제) 비교 가능

### 3.6 가이드 탭

- **카테고리 필터**: 상단 칩 (전체/Push/Pull/Legs/Core/Swing)
- **운동 카드 리스트**: 아코디언 펼침으로 상세
- **상세 내용**: 주동근/보조근, 폼 가이드, 흔한 실수, 안전 주의, 프로그레션 경로, 참고 영상
- **오늘 탭 연결**: 운동 중 운동명 탭 → 가이드 해당 운동으로 직접 이동

### 3.7 디자인 시스템

- **다크/라이트 자동 전환**: CSS `prefers-color-scheme` 미디어쿼리
- **CSS 변수 기반 테마**: 컬러, 타이포, 스페이싱 전부 변수
- **taste-skill 적용**: 프리미엄 타이포그래피, 정밀한 스페이싱, 하드웨어 가속 애니메이션
- **모바일 우선**: breakpoint max-width 700px
- **터치 타겟**: 최소 44px
- **폰트**: Pretendard 또는 시스템 폰트 스택

## 4. Google Apps Script (Code.gs)

### 4.1 API 엔드포인트

```
GET  ?action=init          → exercises + programs(오늘) + user_config 한번에 반환
GET  ?action=programs&month=2026-04  → 해당 월 전체 스케줄
GET  ?action=logs&month=2026-04      → 해당 월 전체 기록
GET  ?action=exercises               → 전체 운동 DB
POST ?action=log           → logs에 세트 기록 추가
POST ?action=config        → user_config 업데이트 (레벨 변경 등)
```

### 4.2 설계 원칙

- 공개 배포 (Execute as: Me, Access: Anyone)
- CORS 허용 (doGet/doPost에서 JSON 반환)
- 에러 시 `{ error: "메시지" }` 반환
- 앱 로드 시 `action=init` 한 번으로 필요한 데이터 전부 로드 (요청 최소화)

## 5. 에이전트 팀 (AGENTS.md)

### 5.1 에이전트 구성 — 7명

| # | 에이전트 | 아이콘 | 핵심 역할 |
|---|---------|--------|----------|
| 1 | 오케스트레이터 | 🎼 | 프로젝트 총괄, 에이전트 배정, 작업 분해 |
| 2 | 운동과학자 | 🧬 | 근비대/근력 원리, 볼륨/빈도/강도 설계, 과학적 근거 |
| 3 | 트레이너 | 🏋️ | 실전 루틴 작성, 종목 선정, 세트/횟수 처방, 월간 스케줄 |
| 4 | 폼 & 안전 전문가 | 🛡️ | 자세 교정, 부상 예방, 흔한 실수, 안전 가이드라인 |
| 5 | 동기부여 코치 | 🔥 | 진급 판정 제안, 볼륨 조절, 심리적 동기, 일지 점검 |
| 6 | 앱 개발자 | 💻 | HTML/CSS/JS, Google Sheets 연동, UI/UX, 디자인 |
| 7 | 커리큘럼 설계자 | 📐 | 5레벨 로드맵, 프로그레션 체인, 분할 구조, 주기화 |

### 5.2 에이전트 간 원칙

- **과학 우선**: 트레이너 처방은 운동과학자 원리에 근거
- **안전 거부권**: 안전 전문가가 과부하/부상 위험 판단 시 스케줄 수정 강제
- **데이터 기반**: 코치 제안은 logs 시트 기록 수치에 근거
- **유저 최종 결정**: 진급, 볼륨 변경 등 모든 변경은 유저 승인 후 반영

### 5.3 핵심 워크플로우

**월간 스케줄 생성**:
```
📐 커리큘럼 → 🧬 운동과학자 → 🏋️ 트레이너 → 🛡️ 안전 → Sheets 기록
```

**월중 조절**:
```
유저 "컨디션 안 좋아" → 🔥 코치 (logs 분석) → 🏋️ 트레이너 (수정안) → Sheets 반영
```

**진급 판정**:
```
🔥 코치 (logs 기반 달성도 확인) → 📐 커리큘럼 (다음 레벨 준비) → 유저 승인 → config 업데이트
```

## 6. 커리큘럼 개요

### 6.1 5레벨 구조

| 레벨 | 이름 | 대상 | 핵심 목표 |
|------|------|------|----------|
| 1 | 기초 체력 | 현재 수준 (푸쉬업 15, 턱걸이 0) | 기본 움직임 패턴 습득, 관절/건 적응 |
| 2 | 기본 근력 | 푸쉬업 25+, 턱걸이 1-3 | 주요 복합운동 수행 가능, 볼륨 점진 증가 |
| 3 | 중급 | 푸쉬업 40+, 턱걸이 8+ | 고급 변형 도입, 볼륨 본격 확대 |
| 4 | 상급 | 고급 변형 수행 가능 | 근비대 집중, 고강도 기법 |
| 5 | 고급 | 고난도 맨몸운동 | 머슬업, 피스톨스쿼트 등 스킬 운동 |

### 6.2 카테고리별 프로그레션 체인 (예시)

- **Push**: 인클라인 푸쉬업 → 스탠다드 → 와이드/다이아몬드 → 디클라인 → 딥스 가중
- **Pull**: 매달리기 → 네거티브 턱걸이 → 턱걸이 → 친업 → 와이드 풀업
- **Legs**: 밴드 스쿼트 → 맨몸 스쿼트 → 불가리안 스플릿 → 피스톨 프로그레션
- **Core**: 플랭크 → 사이드 플랭크 → 행잉 니레이즈 → L-sit 프로그레션
- **Swing**: 케틀벨 데드리프트 → 스윙 12kg → 스윙 16kg → 원핸드 스윙

### 6.3 분할 구조

- **2분할**: A (Push + Core) / B (Pull + Legs)
- **케틀벨 스윙**: A, B 양일 모두 포함 (후면 체인 + 심폐)
- **주간 패턴**: A-B-R-A-B-R-R (주중 집중, 주말 몰아서 가능)
- **시간**: 하루 25-30분 이내

### 6.4 월간 스케줄 운영

- 에이전트팀이 월초 30일분 programs 시트 작성
- 점진적 과부하: 주 단위로 reps +1~2 또는 sets +1
- 디로드 주: 4주마다 볼륨 60% 감소 (회복)
- 유저 요청 시 Claude Code로 해당 주/일 수정

## 8. 범위 외 (YAGNI)

명시적으로 제외하는 항목:

- PWA / ServiceWorker / manifest.json
- 오프라인 캐시
- 자동 타이머 / 알림음
- 소셜 공유 / 커뮤니티
- 사진 기록 (바디 체크)
- 여러 사용자 지원 (1인용)
- 운동 영상 내장 재생 (외부 링크만)
